Usability Test Plan for TeTwix

Usability Test Plan Section

Test Plan for: Web Game TeTwix

Persona Type Tested: Primary, Secondary and Supplemental

Context: The home of the usability subject
Duration: 30—45 minutes

Complete the following before starting: Greeting and Consent & User Background Information.

Overall Objectives
  • Discover errors and areas of improvement;
  • Assess the overall effectiveness of the game TeTwix for different types of users performing the following basic tasks:
    • Initiating a new game;
    • Sharing a score using a social network;
    • Using the various interaction methods supported by the game.
  • Identify the obstacles that may arise in completing the previously mentioned tasks;
  • Evaluate consistency and standards;
  • Evaluate the accuracy with which users complete the tasks;
  • Evaluate the recall of the user regarding the gestural interaction method;
  • Observe the emotional response of the users when playing the game (e.g. do they become frustrated when using a particular input method, do they enjoy playing the game collaboratively, etc.);
  • Evaluate the effectiveness of the user guide;
  • Obtain a list of other social networks that might present interest to users;
  • Obtain suggestions regarding other gameplay modes;
  • Obtain feedback regarding the visual design and possible improvements for it.
Scenario

The user plays a game for the first time using one or more interaction methods. After finishing the game, the user shares his score using the Facebook social network.

The general flow of the scenario is described by the following steps:

  1. The user goes to the web page and opens the game on his laptop;

  2. The application detects and displays all the input devices that are connected, and from which it can receive input;

  3. The user selects the option to start a new game;

  4. A splash screen appears informing the users that he can watch a demo regarding how to play the game;

  5. The user approves and he watches the demo;

  6. After learning how to play the game, they proceed and start controlling tetris pieces;

  7. When the game finally ends, the user is prompted a menu from which he selects the option to share the score using the Facebook social network.

Tasks
  1. Start a new game: 20 seconds;
  2. Play the game using the gestural interface: 5 minutes;
  3. Play the game using the step interface (MakeyMakey): 5 minutes;
  4. Restart the game: 20 seconds;
  5. Pause and then resume the game: 30 seconds;
  6. Play a game until it ends and share the score using the Facebook social network: 1 minute;

    7. A user might complete a task in a shorter or greater time than the estimated time. If the completion time exceeds the estimated time then it there is a problem. Also a user might skip the current task thus the completion time would be 0.

    Post-test Questions

    These questions could be presented to the user as a questionnaire or verbally asked.

    1. What do you think about the color palette used for the tetris pieces (tetriminos)?
    2. What are the aspects that you like regarding the visual design?
    3. What are the aspects that you dislike regarding the visual design?
    4. How would you improve the visual design?
    5. What is the easiest interaction method with the application?
    6. What other methods of interaction would you be interested in?
    7. What other social networks are of interest to you?
    8. What would you change about the gaming experience?
    9. Would you play the game again?
    10. Would you play the game with you family or friends to have fun?
    11. What is the gesture used in order to rotate a piece in the game using the gestural interface?
    12. What is the gesture used in order to move left a piece in the game using the gestural interface?
Notes for Test Coordinator
  • You might explain how usability testing is part of the design and development process. Emphasize that the system is being tested, not them and not you. You are trying to understand how the system would work in their environment. (It's not very helpful if they are trying to imagine how other people would use the system; ask them to test the system for themselves.) It may be helpful to give some examples of how a usability test can improve a system (e.g., show how it gives confusing or contradictory signals, how too many or too few things are supposed to happen at once, or how specific assumptions about the users' knowledge and skills may or may not be true, etc.);
  • Remind the user that you would like them to 'think aloud' so that you can understand what they are thinking, about the task or the product/application;
  • Read the user the scenario for the test;
  • Read the user the tasks he has to complete. If the tasks are complex, you may want to put a sheet of paper in front of the user with each task written out;
  • Record the user's actions, comments, questions, and body language (write them in the Test Observations section accompanied by video/audio);
  • Don't offer help; let the user attempt to perform the task. If they ask for help reply with: "What do you think you/that would do?" "What do you think that means?";
  • Thank the usability subject for taking the time to help you to improve the system. Emphasize how the knowledge you gain from this process will be used to make a more usable system. You might describe how much change is really possible at this point: are you testing an early prototype or a beta release? In other words, can features be added or are you just fixing bugs?

Test Results for: TeTwix

Success Criteria

A successful design has been achieved when:

  • At least 80% of users are able to start a new game and play it collaboratively;
  • At least 60% of users are able to share their score using Facebook;
  • At least 60% of users have a positive emotional response playing the game;
  • At least 50% of users are able to control the pieces of the game using the gestural interface;
  • At least 50% of users understand the idioms of the menu items.

Summary

  • 4/4 users enjoyed the MakeyMakey interaction method which enabled them to control the pieces using their legs;
  • 3/4 users were not able to use the Leap device in order to control the pieces;
  • 2/4 users were a little confused about the meaning of the icons on the menu;
  • 2/4 users found the color palette a bit confusing;
  • 2/4 users had problems predicting the trajectory of the piece; maybe the border is appropriate (A/B testing is required in order to decide which option is better);
  • 1/4 users did not found the Facebook score sharing functionality obvious.

Potential Design Improvements
  • Improve the interaction with the game using the Leap device;
  • Provide multiple colour schemes for the game in order to adapt to different personalities;
  • Provide settings capabilities for the users in order to select the background of their choice when playing: with grid or without;
  • Provide icons with greater semantical value;
  • Provide the user hints in order to take a break from playing the game, since some of the interaction methods may be exhaustive.
